INTRO Cos'è un algoritmo?

La parola algoritmo deriva da un matematico persiano del IX secolo. Ma il concetto è vecchio quanto il pensiero umano: è una sequenza finita di istruzioni per risolvere un problema.

Algoritmo = sequenza di istruzioni
input → [elaborazione] → output
Le 5 proprietà di un buon algoritmo
1

Finitezza

Deve terminare dopo un numero finito di passi. Un algoritmo che gira all'infinito non risolve nulla.

2

Definitezza

Ogni istruzione deve essere precisa e non ambigua. "Aggiungi un po' di sale" non è un'istruzione algoritmica.

3

Input e Output

Prende zero o più input e produce almeno un output. Senza output non c'è risultato.

4

Efficacia

Ogni passo deve essere fattibile concretamente (non "trova il numero più grande dell'universo").

5

Generalità

Deve funzionare per tutti i casi del problema, non solo per un esempio specifico.

Algoritmo della vita reale — la ricetta
🍝 Ricetta = Algoritmo
Una ricetta è esattamente un algoritmo: ha un input (ingredienti), una sequenza di istruzioni (passaggi), e produce un output (il piatto). È finita, definitiva e general-purpose.
🗺 GPS = Algoritmo di ricerca
Quando il navigatore trova il percorso più breve da A a B, sta eseguendo un algoritmo (Dijkstra o A*) su un grafo stradale con milioni di nodi.
🔍 Google = Algoritmo di ranking
PageRank — inventato da Larry Page nel 1998 — assegna un punteggio a ogni pagina web contando i link in entrata. È un algoritmo iterativo che converge in pochi passi.
Pseudocodice: somma di una lista
ALGORITMO SommaLista(A)
// Input: lista A di n numeri
// Output: somma di tutti gli elementi

  somma ← 0
  PER i ← 0 FINO A n-1
    somma ← somma + A[i]
  FINE PER
  RESTITUISCI somma

STORIA 4000 anni di algoritmi

Gli algoritmi non sono nati con i computer. I Babilonesi, Euclide, Al-Khwarizmi, Babbage, Turing — ognuno ha aggiunto un mattone. Il computer moderno è l'ultimo capitolo di una storia lunghissima.

La timeline degli algoritmi
FOCUS Al-Khwarizmi — il padre degli algoritmi

La parola algoritmo deriva dalla latinizzazione del nome Muḥammad ibn Mūsā al-Khwārizmī (780–850 d.C.), matematico persiano di Bagdad. La parola algebra viene dal titolo del suo libro più famoso.

📖 Il Kitāb al-mukhtaṣar
Nel 825 d.C. al-Khwarizmi scrisse il primo trattato sistematico di risoluzione di equazioni lineari e quadratiche. Traduzioni latine medievali introdussero in Europa sia il concetto di algebra che il sistema di numerazione decimale arabo.
🔢 I numeri arabi
Al-Khwarizmi scrisse anche un trattato sul sistema di numerazione indiano (0,1,2...9). Il titolo latino iniziava con "Algoritmi de numero..." — da qui la parola algoritmo.
🕰 Prima di lui: Euclide
Circa 300 a.C., Euclide descrisse il primo algoritmo della storia documentata: il Massimo Comun Divisore. È ancora usato oggi, quasi identico, in crittografia e teoria dei numeri.
// MCD di Euclide (~300 a.C.)
ALGORITMO MCD(a, b)
  MENTRE b ≠ 0
    t ← b
    b ← a mod b
    a ← t
  RESTITUISCI a
FOCUS Ada Lovelace & Alan Turing
👩 Ada Lovelace (1815–1852)
Figlia di Lord Byron, scrisse nel 1843 quello che è considerato il primo programma della storia: un algoritmo per calcolare i numeri di Bernoulli con la Macchina Analitica di Babbage. Riconobbe che la macchina poteva elaborare qualsiasi cosa rappresentabile simbolicamente — non solo numeri.
🧠 Alan Turing (1912–1954)
Nel 1936 pubblicò il paper che definì formalmente cosa significa calcolare. La Macchina di Turing è un modello teorico che può simulare qualsiasi algoritmo. Il Problema della Fermata dimostrò che esistono problemi non risolvibili algoritmicamente — un limite teorico fondamentale.
⚡ La svolta pratica: 1940-1950
ENIAC (1945), il primo computer elettronico generale, eseguiva circa 5.000 operazioni al secondo. Il tuo smartphone ne esegue circa 15 miliardi al secondo. Ma gli algoritmi che girano su entrambi seguono gli stessi principi matematici che Turing aveva descritto nel 1936.
COMPLESSITÀ Quanto è veloce un algoritmo?

Due algoritmi possono risolvere lo stesso problema — ma uno impiega 1 secondo, l'altro 10 anni. La complessità computazionale misura come cresce il tempo (o la memoria) al crescere dell'input.

RICERCA Come trova un algoritmo un elemento?

Due strategie fondamentali: ricerca lineare (controlla uno per uno) e ricerca binaria (dimezza ogni volta lo spazio di ricerca). La seconda richiede che l'array sia ordinato.

🔍 Ricerca Lineare — O(n)
Scorre l'array dall'inizio alla fine. Funziona su qualsiasi array, ordinato o no. Nel caso peggiore guarda tutti gli n elementi.
ALGORITMO RicercaLineare(A, x)
  PER i ← 0 FINO A n-1
    SE A[i] = x
      RESTITUISCI i
  RESTITUISCI -1
⚡ Ricerca Binaria — O(log n)
Confronta con il centro. Se uguale: trovato. Se minore: cerca a sinistra. Se maggiore: cerca a destra. Ogni passo dimezza l'area.
ALGORITMO RicercaBinaria(A, x)
  sx ← 0 ; dx ← n-1
  MENTRE sx ≤ dx
    mid ← (sx + dx) / 2
    SE A[mid] = x → RESTITUISCI mid
    SE A[mid] < x → sx ← mid+1
    ALTRIMENTI → dx ← mid-1
  RESTITUISCI -1

GRAFI Navigare reti e mappe

Un grafo è una struttura fatta di nodi (vertici) collegati da archi (spigoli). Internet, reti sociali, mappe stradali, strutture molecolari — sono tutti grafi.

🌐 BFS — Breadth-First Search
Visita tutti i nodi a distanza 1 dal punto di partenza, poi tutti a distanza 2, poi 3... Come un'onda che si espande. Trova il percorso più breve (in numero di archi).
BFS(grafo G, sorgente s):
  coda ← [s] ; visitati ← {s}
  MENTRE coda non vuota
    v ← coda.estrai_fronte()
    PER u IN vicini(v)
      SE u non visitato
        visitati.aggiungi(u)
        coda.aggiungi(u)
🔎 DFS — Depth-First Search
Esplora il più in profondità possibile lungo ogni ramo prima di tornare indietro. Come un labirinto: segui il filo fino al fondo, poi backtrack. Usa uno stack (o la ricorsione).
DFS(grafo G, nodo v, visitati):
  visitati.aggiungi(v)
  PER u IN vicini(v)
    SE u non in visitati
      DFS(G, u, visitati)

// DFS è naturalmente ricorsivo

RICORSIONE Una funzione che chiama se stessa

La ricorsione è uno dei concetti più eleganti e potenti dell'informatica. Una funzione ricorsiva risolve un problema dividendolo in sottoproblemi più piccoli della stessa natura.

f(n) = caso_base  oppure  f(n-1) + qualcosa
ogni chiamata ricorsiva avvicina al caso base
⚠️ Senza caso base, la ricorsione non termina mai — si genera un overflow dello stack (troppe chiamate annidate). Il caso base è la condizione che "ferma" la discesa ricorsiva.
Esempio 1 — Fattoriale
ALGORITMO Fattoriale(n)
  SE n = 0 // caso base
    RESTITUISCI 1
  ALTRIMENTI
    RESTITUISCI n × Fattoriale(n-1)
5! = 5 × 4! = 5 × 4 × 3! = 5×4×3×2×1×1 = 120
Esempio 2 — Fibonacci
ALGORITMO Fibonacci(n)
  SE n ≤ 1 RESTITUISCI n // caso base
  RESTITUISCI Fib(n-1) + Fib(n-2)
⚠️ Questa versione ha complessità O(2ⁿ) — pessima! Con n=40 fa ~1 miliardo di chiamate. La versione iterativa (o con memoization) è O(n).
Ricorsione vs Iterazione — confronto
📐 Eleganza
La ricorsione spesso produce codice più breve e leggibile per problemi naturalmente ricorsivi (alberi, grafi, divide-et-impera).
💾 Memoria
Ogni chiamata ricorsiva usa spazio nello stack. Per n molto grande rischia stack overflow. L'iterazione usa memoria O(1).
⚡ Velocità
Con memoization (cache dei risultati già calcolati) la ricorsione può essere veloce quanto l'iterazione. Questa tecnica si chiama programmazione dinamica.
